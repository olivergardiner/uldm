IControlledVocabulary extends IReferenceEntity

The IControlledVocabulary Interface is an extension of the IReferenceEntity Interface and simply forces the declaration of a standard <<ReferenceCode>> Property (code), alongside the name and description declared in the super-Interface. As such, it is designed to be used in the very common case where these generic Property declarations are perfectly suitable.

Without further attribution or structure applied to the implementing Class, this Interface creates a simple list of name/code pairs, hence it is considered to represent a simple controlled vocabulary. This is probably sufficient for the vast majority of Reference Data but implementing Classes can also add further structure, as desired (e.g. a hierarchy). 

Note that <<ReferenceEntities>> are expected to be used as the declared Type for Properties. In LDML, this implies that the value of such a property is actually the declared <<ReferenceCode>> (i.e. unlike a DataType, the structure of the Property is not directly instantiated). This concept can be taken to mean that declared Property is instantiated by reference rather than by value.
